# Image Curation

## Princípio central

Use poucas imagens, mas muito bem justificadas.

Independentemente da quantidade de arquivos disponíveis, selecione apenas o que for necessário para sustentar a narrativa. Nunca tente “aproveitar tudo”.

No modo `image-first`, comece amplo para investigar e termine estreito para contar a história. Analisar muitas imagens é permitido. Publicar muitas imagens, não.

## Quando uma imagem merece entrar

A imagem só entra se cumprir pelo menos um destes papéis:

- comprovar uma afirmação do texto;
- mostrar uma mudança relevante de UX/UI;
- esclarecer um fluxo, estado, comparação ou decisão;
- servir como antes/depois realmente informativo.

## Quando descartar

Descarte sem hesitar quando a imagem for:

- redundante com outra já escolhida;
- quase idêntica a outra tela;
- pouco legível;
- ambígua pelo nome e pelo conteúdo;
- bonita, mas sem função narrativa;
- um detalhe irrelevante que o texto já cobre melhor.

## Quantidade recomendada

- Prefira um conjunto enxuto.
- Como ponto de partida, pense em algo entre 4 e 8 imagens relevantes para um case médio.
- Só ultrapasse isso se o case perder clareza sem a prova visual extra.

## Heurísticas de seleção

- Use pares antes/depois quando a comparação for o argumento.
- Vá do macro para o micro.
- Use sequência apenas quando o fluxo realmente exigir múltiplos estados.
- Evite empilhar várias telas da mesma família só porque todas existem.

## Registro mínimo por imagem escolhida

Para cada imagem selecionada, registre:

- arquivo;
- seção em que entra;
- o que a imagem mostra;
- por que ela é necessária;
- qual claim ela sustenta;
- legenda curta e contextual.

## Imagens ambíguas

Se o nome do arquivo não deixar claro o contexto e a imagem não puder ser interpretada com segurança, pergunte ao usuário antes de usá-la como evidência.

## Observação antes de seleção

Quando o insumo principal for imagem, não pule da tela para a galeria final.

Antes de selecionar, registre:

- o que é objetivamente visível;
- o que parece provável, mas ainda é inferência;
- o que depende de confirmação do usuário.

## Anti-padrões

- Galeria inflada.
- Telas repetidas com pequenas variações sem valor argumentativo.
- Screenshots sem legenda.
- Imagem usada só para “preencher” uma seção fraca.
- Transformar análise visual em fanfic de produto.
